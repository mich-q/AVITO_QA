# Тест-кейсы — API Avito QA Internship

**Базовый URL:** `https://qa-internship.avito.com`

## Техники тест-дизайна

| Техника | Где применена |
|---------|---------------|
| Классы эквивалентности | Валидные/невалидные типы для каждого поля (`int` vs `string`, положительное vs отрицательное) |
| Граничные значения | `price=0`, `sellerID=0`, очень большие числа, очень длинные строки |
| Таблица решений | Комбинации отсутствующих обязательных полей (`name`, `price`, `sellerID`) |
| Идемпотентность | Повторные GET-запросы; POST намеренно не идемпотентен |
| Исследовательское | XSS в `name`, SQL injection в параметрах пути, вариации имён полей |

---

## Обнаруженные дефекты

| ID | Описание | Приоритет | Тест |
|----|----------|-----------|------|
| BUG-01 | Отрицательная `statistics` принимается (200 вместо 400) | P1 | TC-CR-13 |
| BUG-02 | Отрицательная цена принимается (200 вместо 400) | P1 | TC-CR-10 |
| BUG-03 | Формат ответа POST не соответствует документации | P2 | TC-CR-02 |
| BUG-04 | Валидный create с нулевой `statistics` отклоняется с 400 | P2 | TC-CR-05 |
| BUG-05 | API принимает недокументированные имена полей и вариации регистра | P2 | TC-CR-15 |
| BUG-06 | API принимает отрицательный `sellerID` | P2 | TC-CR-14 |

Подробное описание каждого дефекта — в файле `BUGS.md`.

## 1. POST /api/1/item — Создание объявления

### Позитивные

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-CR-01 | Валидный полный payload | 200, в ответе UUID объявления |
| TC-CR-02 | Проверка формата ответа | **BUG-03**: возвращается статус-строка, не объект |
| TC-CR-03 | Два одинаковых POST → два разных ID | 200, ID различаются |
| TC-CR-04 | `price = 0` | 200 или 400, но не 500 |
| TC-CR-05 | `statistics = 0/0/0` | **BUG-04**: возвращает 400 (ожидался 200) |

### Негативные

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-CR-06 | Без поля `name` | 400 |
| TC-CR-07 | Без поля `price` | 400 |
| TC-CR-08 | Без поля `sellerID` | 400 |
| TC-CR-09 | Пустое тело `{}` | 400 |
| TC-CR-10 | Отрицательная цена | **BUG-02**: возвращает 200 (ожидался 400) |
| TC-CR-11 | `price` как строка | 400 |
| TC-CR-12 | `sellerID` как строка | 400 |
| TC-CR-13 | Отрицательная `statistics` | **BUG-01**: возвращает 200 (ожидался 400) |
| TC-CR-14 | Отрицательный `sellerID` | **BUG-06**: возвращает 200 (ожидался 400) |
| TC-CR-15 | Недокументированное имя поля, например `sellerId` вместо `sellerID` | **BUG-05**: возвращает 200 (ожидался 400) |

### Граничные / Corner-cases

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-CR-16 | POST не идемпотентен | Два разных ID |
| TC-CR-17 | Очень длинное имя (5000 символов) | 200 или 400, но не 500 |
| TC-CR-18 | XSS в `name` | Не 500 |
| TC-CR-19 | `sellerID = 0` | 400 |
| TC-CR-20 | Граничные значения цены: `0`, `1`, `100000`, `999999999` | Не 500 |

## 2. GET /api/1/item/:id — Получение объявления по ID

### Позитивные

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-GI-01 | Существующее объявление | 200, поля `id/name/price/sellerId/createdAt/statistics` присутствуют |
| TC-GI-02 | Тип ответа — массив | 200, возвращается массив объявлений |
| TC-GI-03 | Данные совпадают с созданными | `name`, `price`, `sellerId` совпадают |
| TC-GI-04 | `statistics` в ответе | Поля `likes/viewCount/contacts` присутствуют |

### Негативные

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-GI-05 | Несуществующий UUID | 404 |
| TC-GI-06 | Невалидный формат ID | 400 или 404 |
| TC-GI-07 | Числовой ID | 400 или 404 |
| TC-GI-08 | Пустой ID | 400/404/405 |

### Corner-cases

| ID | Название | Ожидаемый результат |
|----|----------|---------------------|
| TC-GI-09 | Идемпотентность | 3 одинаковых GET → одинаковые ответы |
| TC-GI-10 | SQL injection в ID | 400/404, но не 500 |

## 4. GET /api/{1,2}/statistic/:id — Статистика

### Позитивные

| ID | Название | Версия | Ожидаемый результат |
|----|----------|--------|---------------------|
| TC-ST-01 | Успешный запрос | v1 | 200 |
| TC-ST-02 | Тип ответа — массив | v1 | 200, возвращается массив statistics |
| TC-ST-03 | Структура ответа | v1 | Поля `likes/viewCount/contacts` |
| TC-ST-04 | Значения совпадают с созданными | v1 | `contacts/likes/viewCount` совпадают |
| TC-ST-05 | Тип данных — int | v1 | `likes/viewCount/contacts` — int |
| TC-ST-06 | Успешный запрос | v2 | 200 |
| TC-ST-07 | Структура ответа | v2 | Поля `likes/viewCount/contacts` |
| TC-ST-08 | v1 и v2 возвращают одинаковые данные | v1 + v2 | Данные совпадают |

### Негативные

| ID | Название | Версия | Ожидаемый результат |
|----|----------|--------|---------------------|
| TC-ST-09 | Несуществующий UUID | v1 | 404 |
| TC-ST-10 | Несуществующий UUID | v2 | 404 |
| TC-ST-11 | Невалидный ID | v1 | 400 или 404 |
| TC-ST-12 | Невалидный ID | v2 | 400 или 404 |

### Corner-cases

| ID | Название | Версия | Ожидаемый результат |
|----|----------|--------|---------------------|
| TC-ST-13 | Идемпотентность | v1 | 3 запроса → одинаковые ответы |
| TC-ST-14 | Идемпотентность | v2 | 3 запроса → одинаковые ответы |
| TC-ST-15 | SQL injection | v1 | 400/404, но не 500 |

## 5. E2E-сценарии

| ID | Название | Шаги |
|----|----------|------|
| TC-E2E-01 | Создать → GET по ID | POST → GET `/item/:id` → данные совпадают |
| TC-E2E-02 | Создать → найти у продавца | POST → GET `/:sellerID/item` → ID в списке |
| TC-E2E-03 | Создать → статистика v1 | POST → GET `/api/1/statistic/:id` → stats совпадают |
| TC-E2E-04 | Создать → статистика v2 | POST → GET `/api/2/statistic/:id` → stats совпадают |
| TC-E2E-05 | Создать 5 → проверить количество | `5x POST` → `len >= 5` |
| TC-E2E-06 | Полный сценарий | POST → GET по ID → список → stat v1 → stat v2 |
